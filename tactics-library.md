# Experience Audit Tactics Library

## Theme 1: Ease of Use
*Can I get things done quickly and correctly?*

### Core Flow
1. Clear navigation and findability
2. Single workspace for key tasks
3. Progressive disclosure (simple → advanced)
4. Smart defaults and pre-filling
5. Low-click flows for common tasks
6. Templates for repetitive work

### Error Prevention
7. Inline validation and helpful constraints
8. Contextual help (tooltips, videos)
9. Drafts and autosave
10. Undo/redo capabilities

### Power Users
11. Keyboard shortcuts
12. Robust search with filters
13. Bulk actions
14. Saved workflows

### Performance
15. Fast feedback on actions
16. Reliable performance at scale
17. Works on poor connections

### Accessibility
18. Strong WCAG support
19. Security without burden (MFA, SSO)
20. Consistent across devices

## Scoring Rubric

**1 = No Evidence Found**
- Extensive public research conducted
- No observable evidence in public materials
- No user reviews mentioning this capability
- List where you looked

**3 = Some Evidence**
- At least one concrete example from public sources
- OR inferred from user reviews/feedback
- Partial implementation or inconsistent application
- Provide specific example

**5 = Strong Evidence**
- Clear, pervasive evidence across multiple public channels
- OR consistently praised in user reviews
- Mature, well-implemented capability
- Multiple examples available

---

## Confidence Levels

**High Confidence:**
- Directly observable in public website, app stores, demo videos
- Can see screenshots, descriptions, or live examples
- First-hand evidence

**Medium Confidence:**
- Inferred from user reviews (Trustpilot, App Store, G2)
- Users explicitly mention this tactic
- Second-hand evidence

**Low Confidence:**
- Inferred from indirect signals (job posts, press releases, partnerships)
- Educated guess based on company type/size
- Proxy evidence

**No Evidence:**
- Extensive search conducted
- No public information found
- Cannot score
